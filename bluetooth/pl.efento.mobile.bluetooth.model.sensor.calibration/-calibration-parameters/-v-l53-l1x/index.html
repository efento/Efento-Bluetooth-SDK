enum TimingBudget : Enum<CalibrationParameters.VL53L1x.TimingBudget>

Timing budget in ms. This is the amount of time the sensor requires to complete a single distance measurement. It can be adjusted to optimize measurement accuracy. Increasing the timing budget reduces the standard deviation of the measurement, resulting in higher accuracy, but also increases power consumption.

crosstalkOffset

val crosstalkOffset: Int

The crosstalk correction value in cps. Range 0 : 65535 This parameter is not required for standard sensor operation. In imaging systems, crosstalk refers to the portion of the return signal detected by the sensing array that originates from internal reflections of the VCSEL light within the protective window (cover glass) placed over the module for protection and aesthetics. The quality of the cover glass can significantly influence the amount of reflected light, potentially impacting sensor performance. The VL53L1x includes an integrated crosstalk compensation mechanism that allows users to mitigate this effect. Crosstalk calibration is used to estimate and apply the necessary correction to compensate for the influence of the cover glass.

val performTemperatureUpdate: Boolean

Indicates if the temperature update should be performed. If the sensor has been inactive for an extended period, it is recommended to perform a temperature update before resuming ranging operations. While the sensor can automatically compensate for temperature changes during continuous operation, a manual temperature update is advised after a prolonged shutdown to ensure accurate performance.
